EmbeddedMQ

Messaging that lives inside your process.

A C11 runtime for queues, pub/sub, and work delivery — no broker, no daemon, no network hop unless you ask for one.

Get started View source

What it is

Link libemq. Create a runtime. Push and pop bytes between threads — with optional durability, backpressure, and ack/nack work queues.

  • In-process first SPSC/MPMC lock-free FAST rings when you stay in one address space.
  • Policies, not frameworks FIFO, LIFO, priority, work, ring, delay, pub/sub — pick per queue.
  • Same ABI, many languages C core with Rust, Python, Go, and Java bindings in the monorepo.

Use it

A minimal FAST queue. Ownership of message.data transfers on pop — release it.

emq_runtime *rt;
emq_queue *q;
emq_queue_opts opts;
emq_message m;

emq_runtime_create(&rt);
emq_queue_opts_default(&opts);
opts.storage = EMQ_STORAGE_FAST;
opts.producers = 1;
opts.consumers = 1;
emq_queue_create(rt, "jobs", &opts, &q);

emq_push(q, "hello", 5, NULL);
emq_pop(q, &m, 1000);
emq_message_release(&m);

emq_queue_close(q);
emq_runtime_destroy(rt);

cmake -S core -B build && cmake --build build && ctest --test-dir build

Monorepo stack

Engine, bindings, and transport planes live side by side.

core/

C11 engine, tests, stress, fuzz, recovery, ABI freeze.

bindings/

Rust, Python, Go, Java clients over the stable C ABI.

transport/

Shared-memory IPC and an Aeron-style network media driver.

Ship with confidence

CI on Windows, Linux, and macOS — plus ASan, UBSan, and TSan. Crash-point recovery, differential tests, and soak suites in the tree.
